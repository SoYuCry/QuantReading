# Session 04 — Chapter 12：Portfolio backtesting 组合回测

阅读范围：Chapter 12 `Portfolio backtesting`

## 1. 本章一句话

第 12 章是全书最重要的实战章之一：

> 机器学习模型给出的只是预测信号；只有把信号转成组合权重，并在严格协议下回测，才能判断它是否有投资价值。

换句话说：

```text
模型好 ≠ 策略好
预测准 ≠ 组合赚钱
历史回测好 ≠ 未来可交易
```

---

## 2. 这一章在全书的位置

前面章节主要回答：

```text
如何用特征预测未来收益？
```

第 12 章回答：

```text
预测出来以后，怎么变成一个真实可检验的投资组合？
```

完整链条是：

```text
特征 X
  ↓
模型预测 y_hat
  ↓
股票排序 / 信号标准化
  ↓
组合权重 w
  ↓
下期真实收益 r
  ↓
组合收益 R_p = sum(w_i * r_i)
  ↓
扣成本、看风险、看换手、看稳定性
```

## 3. Setting the protocol：先定回测协议

回测前必须先明确规则，不能边看结果边改。

至少要定义：

1. **投资 universe**：买哪些股票？例如全部美股、大盘股、流动性过滤后股票。
2. **训练窗口**：用多少历史数据训练模型？固定窗口还是滚动窗口？
3. **预测频率**：月度、周度、季度？
4. **调仓频率**：多久更新一次组合？
5. **持仓数**：买前 10%、前 100 只，还是所有股票加权？
6. **权重规则**：等权、市值加权、按信号强度加权、风险调整权重？
7. **约束**：是否允许做空？单票上限？行业中性？因子中性？
8. **成本假设**：交易成本、滑点、冲击成本。
9. **评估区间**：训练、验证、测试如何按时间分开？

最重要原则：

> 回测协议应该在看最终结果前确定，否则容易产生回测过拟合。

## 4. Turning signals into portfolio weights：信号怎么转权重

机器学习模型通常输出一个预测值：

```text
每只股票的未来收益预测 y_hat_i
```

但投资需要的是组合权重：

```text
每只股票买多少 w_i
```

这中间有几种常见做法。

### 4.1 排序分组法

最常见：

```text
每期按预测收益排序
买预测最高的一组
卖预测最低的一组
```

例如：

- long top 10%
- short bottom 10%
- 或者只做 long-only top 50 stocks

优点：简单、稳健。  
缺点：会丢掉预测值大小信息，只保留排序。

### 4.2 信号比例加权

把预测信号转成权重：

```text
w_i ∝ signal_i
```

比如预测越高，权重越大。

但要注意：

- 极端预测值会导致持仓过度集中；
- 需要 winsorize / clip；
- 需要标准化；
- 可能要行业、市值、波动率中性化。

### 4.3 多空组合

典型 market-neutral 形式：

```text
long 高预测股票
short 低预测股票
long 总权重 = +1
short 总权重 = -1
净暴露 ≈ 0
```

它试图隔离选股信号，减少市场方向影响。

### 4.4 Long-only 组合

如果不能做空，就只能买预测较好的股票：

```text
从 universe 中挑 top N 或 top quantile
```

这更接近实际基金约束，但会更容易混入市场 beta、行业暴露、风格暴露。

## 5. Performance metrics：回测看哪些指标

不能只看累计收益曲线。至少要看以下几类。

### 5.1 收益类

- cumulative return：累计收益
- annualized return：年化收益
- average monthly return：平均月收益

但收益越高不一定越好，因为可能承担了更大风险。

### 5.2 风险类

- volatility：波动率
- maximum drawdown：最大回撤
- downside risk：下行风险

尤其最大回撤很重要，因为它更接近真实持有体验。

### 5.3 风险调整指标

常见：

```text
Sharpe ratio = 平均超额收益 / 收益波动率
```

也可以看：

- Sortino ratio
- information ratio
- alpha / beta

但要注意：

> 高 Sharpe 如果来自过度调参或低估成本，也不可靠。

### 5.4 换手和成本

这是很多漂亮回测死亡的地方。

换手率 turnover 表示组合变动有多频繁：

```text
turnover 高 -> 交易多 -> 成本高 -> 实际收益下降
```

尤其机器学习模型如果每期预测排名变化很大，会导致频繁换仓。

因此必须看：

- gross return：未扣成本收益
- net return：扣成本后收益
- turnover
- capacity：资金规模变大后还能不能成交

## 6. Common errors：常见错误

### 6.1 Forward-looking bias 前视偏差

这是最致命错误之一。

含义：

> 在时间 t 做决策时，使用了时间 t 当时还不知道的信息。

例子：

- 用未来财报数据预测过去；
- 用全样本均值/标准差做标准化；
- 用未来成分股列表构建历史 universe；
- 用未来退市信息过滤股票；
- 标签和特征时间错位。

原则：

```text
任何 t 时刻的特征，都必须是 t 时刻真实可得的信息。
```

### 6.2 Backtest overfitting 回测过拟合

不是模型过拟合，而是研究者把回测系统调到历史最优。

例子：

- 试很多持仓数量，选最好的；
- 试很多调仓频率，选最好的；
- 试很多成本假设，展示最好看的；
- 试很多样本期，只展示有效阶段；
- 不断修改规则直到 Sharpe 好看。

解决方向：

- 预先固定协议；
- 使用真正外样本；
- walk-forward；
- 展示稳健性而非单点最优；
- 少调参，多解释。

## 7. Non-stationarity：非平稳性

金融市场不是静态实验室。

同一个信号可能：

- 在某个时代有效；
- 在另一个时代失效；
- 危机时期表现完全不同；
- 被市场拥挤交易后衰减。

所以要看：

- 分年份收益；
- 不同市场状态下表现；
- rolling Sharpe；
- 因子暴露随时间变化；
- 策略是否依赖少数年份贡献。

关键提醒：

> 回测不是证明未来赚钱，而是排除明显不靠谱策略的一道过滤器。

## 8. 一个完整回测应该怎么讲

一个可信的 ML factor backtest 至少应该报告：

1. 数据来源和 universe；
2. 特征发布日期和滞后处理；
3. 标签定义；
4. 训练/验证/测试时间切分；
5. 模型和超参数选择规则；
6. 信号转权重方法；
7. 调仓频率；
8. 成本假设；
9. 收益、波动、Sharpe、回撤；
10. 换手率；
11. long leg / short leg 分解；
12. 因子暴露和 alpha；
13. 分时期稳健性；
14. 失败阶段分析。

如果只给一张漂亮净值曲线，不够。

## 9. 和第 10 章的连接

第 10 章讲：

```text
模型预测是否泛化？
```

第 12 章讲：

```text
泛化的预测是否能变成可交易组合？
```

两者缺一不可。

一个模型可能：

- RMSE 不错，但排序能力差；
- accuracy 高，但赚小亏大；
- top quantile 有收益，但换手过高；
- gross return 好，但 net return 差；
- 总回测好，但只靠某几年。

## 10. 读这一章时的核心判断框架

以后看到任何量化策略回测，按这个顺序问：

```text
1. 数据有没有泄漏？
2. 训练/测试是不是时间外样本？
3. 信号怎么转成权重？
4. 有没有扣成本？
5. 换手率多高？
6. 收益来自 long 还是 short？
7. 是否只是市场/行业/风格暴露？
8. 不同年份是否稳定？
9. 是否经过过度调参？
10. 失败时会亏多少？
```
